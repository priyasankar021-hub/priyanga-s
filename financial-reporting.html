The Problem

Every single working day, someone on the team had to produce a personalised sales pipeline report for each sales leader — from scratch. It sounds manageable. In practice, it consumed two hours of skilled time before the real work of the day could even begin.

The pain was the relentless, precision-demanding repetition — pull the CRM dump, clean it, rebuild the pivots, compare manually against yesterday's numbers, colour-code every movement by hand, write up the analysis, then re-filter and re-send individually to each sales owner and leader. Every. Single. Day.

The Solution

The solution is a three-layer pipeline built entirely within Excel — Power Query for ETL, Power Pivot for data integration and comparison, and Excel VBA for personalised email generation and Outlook automation. Zero new software, zero additional cost, and fully operable by anyone on the team.

The key design principle: the user's daily input is reduced to two actions — drop in the raw files, then select a stakeholder name and run. Everything else — cleaning, comparing, formatting, filtering, and emailing — happens automatically.
01
Drop in today's and yesterday's CRM exports
The user downloads two raw data dumps from the CRM — today's pipeline and yesterday's pipeline — and drops them into the designated input location in the working file. That's the only manual data step. No formatting, no cleanup, no copy-pasting required.
CRM Raw ExportToday's DataYesterday's Data
02
Power Query runs the full ETL process
Power Query automatically ingests both files and runs a saved set of transformation steps — removing duplicates, standardising column names and data types, fixing date formats, handling nulls, and restructuring the layout. Both files are cleaned and shaped consistently on every refresh, with no manual intervention. The same logic that took 30 minutes by hand now runs in seconds.
Power QueryETL PipelineAuto-clean StepsDual-file Ingestion
03
Power Pivot integrates both datasets and computes variances
Power Pivot loads the cleaned outputs from both files into a unified data model and runs the day-on-day comparison logic — calculating movement in number of opportunities, total contract value, revenue, deal stage, close date, probability, and win/loss status. The comparison that used to require opening two files side by side and computing manually, cell by cell, now happens in the data model automatically.
Power PivotData ModelDay-on-Day VarianceMulti-dimension Comparison
04
Ten pivot tables auto-populate with compared data
Ten pivot tables — covering New Opportunities, TCV Changes, Revenue, Sales Phase, Close Date, Probability, Deal Won, Deal Lost, Pipeline summary, and Large Deals — all draw from the Power Pivot model and refresh automatically. Conditional formatting highlights movement in green (up) or red (down) across every dimension. The manual colour-coding step is gone entirely.
Excel Pivot TablesPower Pivot ModelAuto Conditional Formatting10 Report Dimensions
05
Select the stakeholder — VBA does the rest
The user selects a sales region or leader name from a slicer and runs the VBA macro. The macro loops through each stakeholder in the slicer, applies the filter for that person's portfolio, creates a new Outlook email, and embeds all ten formatted pivot tables directly into the email body — using the Word editor for precise formatting. Each email is personalised to that leader's data only. If a pivot table has no changes for a given recipient, the macro inserts "No changes as of today" automatically.
Excel VBASales Org SlicerOutlook AutomationWord Editor IntegrationPer-recipient Filter Loop
06
User verifies and clicks send
Each email opens in Outlook as a draft, ready for review. The user can verify the data, make any last-minute edits, and click send. The entire distribution — multiple personalised emails, each filtered to the right leader — is handled in a single macro run. What used to be 20 minutes of re-filtering and re-sending, one by one, is now a human review step only.

The Impact

The headline number — 460+ hours saved annually — translates to more than 11 full working weeks returned to the team every year. But the qualitative shift matters just as much: every sales leader now receives a consistent, accurate, personalised report before the day starts, with no human effort required.

Key Outcomes
  • 460+ man-hours saved per year — 11+ working weeks of skilled time recovered
  • 90% reduction in time per daily reporting cycle (2 hours to 5 minutes)
  • Eliminated manual errors from colour-coding, comparison, and copy-pasting
  • Each sales leader receives a personalised, portfolio-filtered report automatically
  • Day-on-day analysis is consistent and comparable, not subject to human variation
  • Team freed to spend time on analysis and strategy rather than repetitive data work
