01 The Problem

Every working day, someone on the team had to produce a personalised sales pipeline report for each sales leader — from scratch. Before any strategy work could begin, two hours were already gone to a repetitive, error-prone manual process.

1 Export raw data from the CRM 10 min ▾
Manually log into the CRM, run the pipeline export and download the raw file. The format was inconsistent every time — different column orders, missing fields, varied date formats — making every export a fresh challenge.
2 Clean and reformat the export 20 min ▾
Remove duplicates, fix broken columns, standardise date and value formats, and restructure the layout so it's usable for analysis. The same cleanup issues appeared every day, requiring the same manual fixes every time.
3 Extract multiple pivot tables manually 20 min ▾
Build or refresh pivot tables for number of opportunities, deal values, pipeline stage distribution, and sales owner breakdown. Each one manually rebuilt or refreshed from the cleaned data — no automation, no shortcuts.
4 Compare with yesterday's data side by side 25 min ▾
Open the previous day's report alongside today's. Go through every metric manually — did opportunities increase or decrease? By how much? Did the total pipeline value move up or down? All computed cell by cell with no formula logic to help.
5 Colour-code every movement by hand 10 min ▾
Highlight cells green where numbers went up, red where they went down. No conditional formatting — every colour applied manually. Miss one cell and the report gives a sales leader the wrong signal.
6 Write up the analysis and compose emails 15 min ▾
Summarise the day's movements in plain language — "Opportunities increased by X, pipeline value down by Y%" — then manually compose the email with the analysis and data pasted in. Written from scratch every day.
7 Re-filter and send individually to every leader 20 min ▾
Each sales owner and leader received their own version — filtered to their specific portfolio only. The entire report had to be re-filtered and a separate email composed and sent for every single recipient, one by one. This was the most painful part of the whole process.

02 Solution Approach

The solution is a three-layer pipeline built entirely within Excel — no new software, no additional cost, and fully operable by anyone on the team.

Design principle: the user's daily input is reduced to two actions — drop in the raw files, then select a stakeholder name and run. Everything else — cleaning, comparing, formatting, filtering, and emailing — happens automatically.
1 Drop in today's and yesterday's CRM exports manual ▾
The user downloads two raw data files from the CRM — today's pipeline and yesterday's — and drops them into the designated folder. That is the only manual data input required.
CRM ExportFile Input
2 Power Query runs the full ETL automatically auto ▾
Power Query connects to both files and runs a saved set of transformation steps — removing duplicates, standardising column names and data types, fixing date formats, handling nulls, and restructuring the layout. Both files are cleaned consistently on every refresh with no manual intervention.
Power QueryETL PipelineAuto-clean Steps
3 Power Pivot integrates both datasets and computes variances auto ▾
Power Pivot loads the cleaned outputs from both files into a unified data model and runs the day-on-day comparison — calculating movement in number of opportunities, total contract value, revenue, deal stage, close date, probability, and win/loss status. The side-by-side manual comparison is completely eliminated.
Power PivotData ModelDay-on-Day Variance
4 10 pivot tables auto-populate with colour-coding auto ▾
Ten pivot tables covering New Opportunities, TCV Changes, Revenue, Sales Phase, Close Date, Probability, Deal Won, Deal Lost, Pipeline summary, and Large Deals — all draw from the Power Pivot model and refresh automatically. Conditional formatting highlights movements in green or red with no manual colouring.
Pivot TablesAuto Conditional Formatting10 Dimensions
5 Select stakeholder from slicer, run the macro manual ▾
The user selects a sales region or leader name from a slicer and runs the VBA macro. One selection, one click.
Excel SlicerVBA Trigger
6 VBA generates personalised emails for every leader auto ▾
The macro loops through each sales owner and leader, filters the report data to each person's specific portfolio, generates their personalised summary, and creates individual Outlook emails — each containing only their data, with all ten pivot tables embedded. If a pivot table has no changes for a given recipient, it automatically inserts "No changes as of today."
Excel VBAOutlook AutomationPer-recipient Filter Loop
7 User verifies once and clicks send manual ▾
Each email opens in Outlook as a ready-to-send draft. The user reviews the content, makes any last edits, and clicks send. Human review is preserved — everything else has already been done automatically.

03 Impact

The shift from manual to automated reporting eliminated error risk at every step and returned significant time to the team — time that now goes into analysis and strategic work rather than repetitive data handling.

460+ man-hours saved per year — equivalent to 11+ full working weeks
Daily reporting time reduced from 2 hours to 5 minutes — a 90% effort reduction
Eliminated manual errors from colour-coding, comparison, and copy-pasting
Each sales leader now receives a personalised, portfolio-filtered report automatically
Day-on-day analysis is consistent and comparable — not subject to human variation
Team time freed for higher-value strategic analysis and stakeholder engagement
